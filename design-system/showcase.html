// PHILOSOPHY
01 / PHILOSOPHY

Five rules for every component

These rules are non-negotiable. If a component doesn't pass all five, it doesn't ship.

RULE 01

Paired shadows

Every raised surface has a light highlight on the top-left and a dark shadow on the bottom-right. That's what makes it clay, not just a card.

RULE 02

Glossy highlight

Icons and badges carry a thin white "moisture line" at the top edge. It costs three lines of CSS and adds the entire "pressed" quality.

RULE 03

Generous radii

Minimum 12px on small surfaces, 18–24px on cards, 999px on pills. Sharp corners are forbidden — they break the soft, friendly tone.

RULE 04

Motion with intent

Every transition is 200–500ms with cubic-bezier(0.16, 1, 0.3, 1). No instant state changes. No bouncy springs on body text.

RULE 05

Purposeful color

Cyan = primary. Green = success / safe. Amber = warning. Red = danger. Ink is for text only. The palette never decorates — it communicates.

// MOTION
07 / MOTION

Curves and durations

SoftClaw uses a single easing curve and three duration tiers. That's it. No bouncy springs, no instant cuts, no 2-second flourishes.

FAST

180ms

For micro-interactions: button press, hover state, mark toggle.

MEDIUM

280ms

For tab switches, card reveals, page transitions.

SLOW

480ms

For hero gauge fills, page load reveals, key visual moments.

EASING
cubic-bezier(0.16, 1, 0.3, 1) /* ease-out — the only curve */

This is the "Apple-style" out-cubic: it starts fast and decelerates smoothly. It's the only curve used in the entire system. One curve = consistent feel.
